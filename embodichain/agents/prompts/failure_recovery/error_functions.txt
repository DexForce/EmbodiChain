1. `misplaced_object`
Category:
- `object_error_types`

Purpose:
- Shift a rigid object away from its expected pose.

Key arguments:
- `error_obj`: target rigid object name, such as `"cup"` or `"bottle"`. If not provided, one non-table rigid object may be selected
- `error_pose`: optional absolute target pose
- `relative_error_xyz`: optional xyz offset such as `[0.03, -0.02, 0.0]`

Typical failure meanings:
- target object shifted before grasp
- placed object no longer at the expected location

2. `fallen_object`
Category:
- `object_error_types`

Purpose:
- Displace and tip over a rigid object.

Key arguments:
- `error_obj`: target rigid object name
- `error_pose`: optional absolute target pose
- `relative_error_xyz`: optional xyz offset before falling is applied

Typical failure meanings:
- affordance becomes invalid
- object can no longer be grasped or aligned as planned

3. `wrong_affordance`
Category:
- `action_error_types`

Purpose:
- Perturb the acting arm trajectory toward a wrong target pose / affordance.

Key arguments:
- `error_arm`: `"left_arm"` or `"right_arm"`
- `relative_error_xyz`: optional perturbation such as `[0.02, 0.02, 0.01]`

Typical failure meanings:
- off-center grasp
- inaccurate placement
- poor approach pose

### Atomic Graph Actions for AgentTaskGraph

Agent edges must use structured atomic graph action specs, not Python function
calls. Do not output `{"fn": ...}` as an arm action. Each compiled action plans
to the standard atomic IO:

`(is_success, trajectory, joint_ids)`

The first version supports only these top-level action names:

- `move`
- `pick_up`
- `place`
- `gripper_open`
- `gripper_close`

Common action schema:

```json
{
  "kind": "atomic_action",
  "name": "<move|pick_up|place|gripper_open|gripper_close>",
  "cfg": {
    "control_part": "<left_arm|right_arm|left_eef|right_eef>"
  },
  "target": {
    "kind": "<target kind>"
  },
  "runtime_kwargs": {}
}
```

Use `left_arm_action` and `right_arm_action` fields on graph edges. Put `null`
for an idle arm.

## `pick_up`

Purpose: move to the target object's grasp pose, close the gripper, and lift the
object.

Schema:

```json
{
  "kind": "atomic_action",
  "name": "pick_up",
  "cfg": {
    "control_part": "right_arm",
    "hand_control_part": "right_eef",
    "pre_grasp_distance": 0.1
  },
  "target": {
    "kind": "object_semantics",
    "obj_name": "bottle"
  },
  "runtime_kwargs": {}
}
```

## `place`

Purpose: move above the placement pose, descend to the release pose, open the
gripper, and retract.

Schema:

```json
{
  "kind": "atomic_action",
  "name": "place",
  "cfg": {
    "control_part": "right_arm",
    "hand_control_part": "right_eef",
    "lift_height": 0.08
  },
  "target": {
    "kind": "table_pose",
    "obj_name": "bottle",
    "x": 0.1,
    "y": 0.5
  },
  "runtime_kwargs": {}
}
```

Use `place` when the arm is already holding the object and the desired result is
to release it on the table.

## `move`

Purpose: move the arm end-effector or arm joints without directly controlling
the gripper. All nominal relative movement, absolute movement, return-home,
orientation, and pouring wrist rotation should be represented by `move`.

Required arm cfg:

```json
{
  "control_part": "right_arm"
}
```

Supported target kinds:

- `pose`, `matrix`, or `target_pose`: provide a 4x4 target pose matrix.
- `object_relative_pose`: fields `obj_name`, optional `x_offset`, `y_offset`,
  `z_offset`; preserves the current orientation.
- `absolute_position`: optional `x`, `y`, `z`; omitted coordinates stay at the
  current end-effector position.
- `relative_offset`: optional `dx`, `dy`, `dz`, optional `mode` of `extrinsic`
  or `intrinsic`.
- `initial_pose`: return the arm to its initial joint configuration.
- `eef_orientation`: field `direction`, either `front` or `down`; keeps the
  current end-effector position.
- `joint_delta` or `eef_rotation_delta`: fields `joint_index` and `degree` (or
  `radian`). Use this for pouring or tilting, for example
  `{"kind": "eef_rotation_delta", "joint_index": 5, "degree": -90}`.

Examples:

```json
{
  "kind": "atomic_action",
  "name": "move",
  "cfg": {"control_part": "right_arm"},
  "target": {
    "kind": "object_relative_pose",
    "obj_name": "cup",
    "x_offset": 0.05,
    "y_offset": 0.1,
    "z_offset": 0.1
  },
  "runtime_kwargs": {}
}
```

```json
{
  "kind": "atomic_action",
  "name": "move",
  "cfg": {"control_part": "right_arm"},
  "target": {
    "kind": "eef_rotation_delta",
    "joint_index": 5,
    "degree": -90
  },
  "runtime_kwargs": {}
}
```

```json
{
  "kind": "atomic_action",
  "name": "move",
  "cfg": {"control_part": "right_arm"},
  "target": {"kind": "initial_pose"},
  "runtime_kwargs": {}
}
```

## `gripper_close` / `gripper_open`

Purpose: execute a gripper-only trajectory.

Schema:

```json
{
  "kind": "atomic_action",
  "name": "gripper_close",
  "cfg": {
    "control_part": "right_eef",
    "arm_control_part": "right_arm"
  },
  "target": {
    "kind": "gripper_state",
    "state": "close"
  },
  "runtime_kwargs": {}
}
```

```json
{
  "kind": "atomic_action",
  "name": "gripper_open",
  "cfg": {
    "control_part": "right_eef",
    "arm_control_part": "right_arm"
  },
  "target": {
    "kind": "gripper_state",
    "state": "open"
  },
  "runtime_kwargs": {}
}
```

Do not invent action names such as `pour`, `tilt`, `rotate_eef`,
`orient_eef`, `back_to_initial_pose`, or `upright_object`. Express those
intentions with `move` target kinds when they are supported.

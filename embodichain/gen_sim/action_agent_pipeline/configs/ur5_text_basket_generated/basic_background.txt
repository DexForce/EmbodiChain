The scene comes from the exported 1779900000_gym_project mesh environment.

This configuration directory is for the UR5BreadBasket task template. The
current robot is a dual-UR5 composite robot with DH_PGI_140_80 parallel
grippers.

The robot is a dual-UR5 composite robot with two parallel grippers:
- left_arm is the UR5 outside the left side of the table's near long edge.
- right_arm is the UR5 outside the right side of the table's near long edge.

Both UR5 bases are on the same long side of the table and face inward toward
the central wicker_basket. The bases are intentionally kept
outside the table edge to avoid initial robot-table contact.

The interactive objects are:
- left_apple: the apple mesh initially on the
  negative-x side (source object apple_2).
- right_apple: the apple mesh initially on the
  positive-x side (source object apple_1).
- wicker_basket: the target container near the center of the
  table (source object basket_3).

The nominal task starts with simultaneous dual-arm grasping. The left UR5 must
grasp left_apple while the right UR5 grasps
right_apple in the same graph edge. After both
apples are held, placement is sequential: the left UR5 places
left_apple into wicker_basket and returns
to its initial pose, then the right UR5 places right_apple
into wicker_basket and returns to its initial pose. To change
the insertion order later, edit the task prompt sequence and keep the same
atomic action API.

The wicker_basket area is a shared workspace. After a UR5
releases a target object, it should retreat upward and return to its initial
pose before the other UR5 moves to the container, otherwise the two arms may
collide near the container. The right UR5 should keep holding
right_apple while the left UR5 performs its placement and
return-to-initial sequence.

A target object at a high pose above `wicker_basket` is only
staged, not placed. Each arm must lower the held object into the container
release pose and open the gripper before any return-to-initial motion.

Always plan to the current `wicker_basket` object pose from the
environment config. Do not hard-code container coordinates in generated graph
actions.
